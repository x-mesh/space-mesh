# space-mesh Design Context

## Design Context

### Users
개발자 본인(macOS 파워유저). 디스크 여유공간이 부족할 때 — 주로 저녁/야간 작업 중 — 열어서 "어디가 큰지 확인 → 검증 → 회수"를 몇 분 안에 끝내려는 도구. 신뢰가 최우선: 지워도 되는 것과 아닌 것이 한눈에 구분되어야 한다.

### Brand Personality
정밀한 · 기계적인 · 신뢰할 수 있는. 항공기 계기판/전문 계측 장비의 감성 — 숫자가 주인공이고, 장식은 없고, 모든 표시에는 이유가 있다. 화려함보다 밀도와 정확함으로 인상을 남긴다.

### Aesthetic Direction
- **테마**: 다크 기본 (야간 작업 + treemap 데이터 색이 다크에서 선명). 순수 검정 금지 — 앰버 쪽으로 틴트된 웜 다크 뉴트럴.
- **색**: UI 크롬은 모노톤. 악센트는 웜 앰버 단 하나 (계기판 바늘/눈금 연상, "공간 회수" 행위와 어울리는 온도). 컬러는 데이터(treemap 타일)가 담당 — 60/30/10.
- **타이포**: 데이터/숫자는 monospaced(tabular) 리드아웃, 섹션 라벨은 대문자 + 넓은 자간(계기판 라벨). 플랫폼 제약상 시스템 폰트의 design variant로 구현.
- **안티레퍼런스**: 알록달록한 "클리너" 앱들(CleanMyMac류의 마케팅 감성), 글래스모피즘, 그라디언트 텍스트, 네온 다크.

### Design Principles
1. 숫자가 주인공 — 크기·개수는 항상 monospaced digit, 정렬은 우측.
2. 색은 의미가 있을 때만 — 앰버(주목/액션), 그린(안전), 오렌지(검토), 나머지는 데이터의 색.
3. 장식 없음 — 모든 시각 요소는 상태나 위계를 전달해야 존재 가능.
4. 파괴적 행위일수록 조용하게 — 삭제 버튼은 눈에 띄되 요란하지 않게, 확인은 명확하게.
5. 밀도를 두려워하지 않기 — 전문가 도구답게 정보 밀도를 유지하되 spacing 리듬으로 그룹화.
